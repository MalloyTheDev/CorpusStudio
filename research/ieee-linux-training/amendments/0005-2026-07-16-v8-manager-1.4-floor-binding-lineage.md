This amendment is **append-only**. It does not edit `PROTOCOL.md`, `EXPERIMENT_MATRIX.yaml`, amendments
0001-0004, effective matrices 1.1.0-1.4.0, or `RESERVED_IDENTITIES.v1`..`.v4` in place. It adds effective

`supersedes` block, so the amendment chain stays ordered and 0004 stays provably byte-frozen.

## Why this amendment exists

The v7 0.5B bring-up (effective matrix 1.4.0, worker wheel
`090f879b46d52e8c33c96fad8aeb61a41c320d44c20bd189dabfd5be606479b2`, worker source `21aa81d97ff752709fd4d03791288c1bb76a2339`)
produced **`V7_MATH_AND_FLASH_THROUGHPUT_PASS`**: both matched arms completed 12 QLoRA optimizer steps
with decreasing loss, admitted a clean adapter, measured `NATIVE_SAFE`, and - unlike v6 - recorded
**valid** token throughput (math `run-019f6956-d55c-7684-8c7a-9ebb7bad7a04` ~104 tok/s, flash
`run-019f6966-1c87-71e4-8c30-6fc5fa085caf` ~108 tok/s), with `scientific_throughput_complete=true` on
both arms. Those v7 identities are now fully instantiated and are reserved here.

Two subsequent, merged control-plane corrections force a fresh v8 wheel and lineage:

1. **Exact per-lineage source-floor binding (PR #471, merged `4bab1de6a844acfb7443344880c5a82c1ac8e872`).**
   `required_git_ancestor` is now an exact, per-lineage plan input supplied to
   `env-plan --required-git-ancestor`, sealed into the `DependencyResolution`/confirmation hash, admitted
   by byte-equality against the worker wheel's EMBEDDED floor at `env-create`, and recorded in the
   `EnvironmentLock`. It is not a recipe constant and not a permanent historical minimum. The v8 wheel
   must EMBED this exact floor and be built with `build-worker-wheel --required-ancestor 4bab1de6a844acfb7443344880c5a82c1ac8e872`.
2. **Build-provenance `source_commit` emission (the v7 follow-up).** The worker build-provenance
   generator must emit the canonical `source_commit` key (v7 embedded it only under a prohibited alias).
   A v8 wheel's embedded `BUILD_PROVENANCE.json` therefore differs from v7's, changing the wheel
   content hash and forcing a fresh wheel identity.

## Environment Manager 1.4.0 (the version decision)

The Environment Manager mints one minor version per additive lock/evidence generation: 1.0.0 (introduce)
-> 1.1.0 (flash timing/thermal evidence) -> 1.2.0 (installed-file tree digest) -> 1.3.0 (successful-run
evidence). PR #471 added `required_git_ancestor` to the `EnvironmentLock` (a new additive lock field) and
changed worker-recipe admission behavior. Under the established convention this warrants **1.4.0**, and
this amendment declares `environment_admission.manager_version_exact = "1.4.0"` so the sealed matrix and
the implementation move together (the tested `manager_version_exact == MANAGER_VERSION` invariant). The
change is provably additive: `recipe_digest`, `resolution_digest`, and `_lock_digest` pop the field when
null, so every non-worker recipe and every pre-existing lock/resolution retains a byte-identical seal.
Historical matrices 1.1.0-1.4.0, amendments 0001-0004, `RESERVED_IDENTITIES.v1`..`.v4`, and all legacy
manager-1.3 locks/environments/evidence remain unchanged historical artifacts; manager 1.4.0 does not
rewrite or reclassify any of them.

## Reserved identities

`RESERVED_IDENTITIES.v5.json` is an append-only strict superset of v4. Beyond every v1-v6 identity v4
carried, it reserves the fully instantiated **v7** identities: the v7 worker wheel (`090f879b46d5...`)
and source commit (`21aa81d97ff7...`); the `math-v7`/`flash-v7` environment ids and lock

v1-v7 identity may be reused; the fresh v8 pair (`math-v8`/`flash-v8`) is disjoint from all of them.

## Seven-B native-Linux feasibility ladder (separate non-paper arm)

This amendment defines a separate, non-paper feasibility arm `seven_b_native_linux_feasibility_ladder`
over `Qwen/Qwen2.5-7B-Instruct` against an immutable small license-clear chat fixture (the private corpus
is untouched; no truncation). It walks sequence-length rungs 512, 1024, 2048, 3072, 4096; per authorized
rung it runs one bounded 12-step feasibility run with microbatch 1 and gradient accumulation 1, math
first and flash conditional (an unexecuted flash rung is `NOT_RUN`, never a pass), no offload, and any
failed fit is retained honestly. **Feasibility runs are explicitly not primary paper cells and do not
satisfy the three-trial characterization matrix.** Sequence-length-4096 success requires a real completed
run, positive token evidence, changed adapter state, an admitted artifact, complete telemetry, and a
measured fit - nothing weaker. This amendment authorizes only the preregistration; it downloads no model
and executes no rung.

## Preregistration boundary (unchanged)

Feasibility characterization remains distinct from the gated full-training phase over the ~500-output
corpus; that phase is not authorized here. No 7B model is loaded, no wheel is built, no environment is
created, and no rung is executed by this amendment.
